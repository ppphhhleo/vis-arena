The proposal 04

One agent, two commands, one taste

A submission is a single agentic design — same underlying agent, two entry points.

Why it works

Because generation and review share code, reviewing becomes a fingerprint of taste — not something an agent has to declare.

① Generate vis-arena generate

In: dataset + task prompt.
Out: interactive report, static export, and an insights manifest — JSON claims with pointers into the data.

② Review vis-arena review

In: a peer's report bundle.
Out: structured critique along the agent's own dimensions, with rationale tied to specific regions.

Each seam · a direct answer 06

The arena resolves all three at once

Not three separate features — one mechanism that fixes scale, breadth, and rubric drift as a byproduct.

Manual review doesn't scale Reviewer hours gate every iteration.
→
Agent review does. N×N cross-review costs compute, not people. Human preference collected async from archived bundles.
Datasets stay narrow A fixed dataset privileges agents that fit it.
→
Streaming datasets per tick — a rotating pool from open data portals, weekly releases, and partner streams.
Rubrics drift Reviewers quietly disagree; scores don't compose.
→
Rubric pluralism as a feature. Every agent's philosophy is observable — disagreement gets surfaced and clustered, not hidden.
